# 发布说明

本仓库的公开版本采用“干净历史 + 脱敏源码 + 本地真实配置”的发布策略。

## 发布内容

Release 包含：

- `南山/` 源码、前端模板、启动脚本、样例配置和子项目 README。
- `青年驿站/` 命令行脚本、Web 控制台、样例配置和子项目 README。
- `docs/` 下的用户教程、项目动机、困难复盘、系统实现和本发布说明。
- 根目录 `README.md`、`.gitignore`、`LICENSE`。

Release 不包含：

- `config.json`
- `南山/JSON/`
- Cookie、JWT、代理接口、通知密钥、身份证号、手机号等真实配置
- 虚拟环境、Conda 包、二进制包、日志、缓存、博客改写材料、图片素材和实验脚本

## 恢复方式

1. 从 GitHub 拉取公开仓库。
2. 按 `README.md` 和 `docs/USER_GUIDE.md` 安装依赖。
3. 将本地安全保存的真实配置复制回 `南山/config.json` 或 `青年驿站/config.json`。
4. 重新抓取目标平台登录态，确认 Cookie/token 仍然有效。
5. 先做登录态/预检验证，再启动正式任务。

## 为什么不保留旧 Git 历史

旧私有开发历史里可能包含临时 token、代理接口、通知地址或测试配置。公开仓库如果保留这些提交，即使当前文件已经脱敏，历史仍然可以被下载和检索。

因此公开归档版使用干净历史发布，只保留最终整理后的工程形态和复盘文档。旧历史只适合本地留存，不适合 Public 仓库。
